## üßæ **1Ô∏è‚É£ What IGEMS Barcodes Are**

In IGEMS (the CAD/CAM suite for waterjet cutting), **barcodes are metadata identifiers** embedded into job reports, setup sheets, or printed NC program summaries.

They‚Äôre typically **Code 39**, **Code 128**, or **QR** symbols that encode a **unique job reference**, which corresponds to:

* a **CNC file**,
* a **drawing/project**, or
* a **nest ID (sheet layout)**.

This barcode lets operators or production managers quickly identify and pull up the digital job data.

---

## üéØ **2Ô∏è‚É£ Purpose of Barcodes in IGEMS Workflow**

| Use Case                    | Description                                                                                                                                      |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Job Identification**      | Each barcode uniquely identifies a job or sheet. Operators scan it to open the matching NC file or production data instantly.                    |
| **Automation**              | Used in smart factories or MES (Manufacturing Execution Systems) ‚Äî scanning triggers machine setup scripts or production tracking.               |
| **Traceability**            | Links physical parts/sheets to their corresponding cutting files, material batches, or customers.                                                |
| **Paper-to-Digital Bridge** | When operators receive printed IGEMS reports on the shop floor, scanning the barcode loads the corresponding job on their tablet or workstation. |
| **Error Reduction**         | Avoids manual job selection errors (especially when dozens of NC files exist for different sheets).                                              |

---

## üß© **3Ô∏è‚É£ Where IGEMS Adds Barcodes**

IGEMS lets you add barcodes via the **Report Template Editor**:

* In `Reports ‚Üí Job Report Template ‚Üí Edit Template`.
* You can insert a **Barcode Field** (linked to variables like `JobName`, `FileName`, or `SheetName`).

Typical placements:

* Top-right corner of job report.
* Next to sheet info or file list.

### üß† Example:

A barcode could represent:

```

File: /nc/WJ20251101-SS30.nc
```

Encoded as text:
`WJ20251101-SS30`

When scanned, the operator‚Äôs tablet app receives this Job ID and fetches all related info.

1. You can insert a barcode object (similar to adding a text field).

   * **Barcode type:** Code 39, Code 128, QR, etc.

### üîπ **Text / Datasource Field**

* The key setting here is the **Datasource ‚Üí Field** dropdown (on the right of your screenshot).
* It allows you to pick *which IGEMS variable* will be encoded into the QR code.

From your image, you can see options like:

```

```

Each of these categories expands to **subfields**, e.g.:

* `Job.CNCFile`

* `Material.Name`
* `SheetReport.SheetName`
* `Machine.Name`
* etc.

## üß† **How to Use It for Your Integration**

You can make this barcode encode **any combination** of fields ‚Äî that‚Äôs what your web app will later read.

### ‚úÖ **Option 1 ‚Äî Encode a Single Identifier**

Simplest and cleanest:

* Field: `SheetReport.SheetName`
  ‚Üí e.g. `SHEET-3670`

Then your tablet app scans the QR ‚Üí sends request:

```
GET /api/jobs/SHEET-3670
```

---

### ‚úÖ **Option 2 ‚Äî Encode Multiple Fields (recommended for automation)**

You can also **concatenate** fields to embed richer info (job name, material, thickness, etc.) in one QR.

Example (set ‚ÄúText‚Äù to):

```
=Job.Name & "|" & Material.Name & "|" & Material.Thickness & "|" & SheetReport.SheetName
```

This QR will encode something like:

```
SMB Eng 30mm SS|Stainless Steel|30|SHEET-3670
```

Your Android app can then decode and parse it easily into:

```json
{
  "job": "SMB Eng 30mm SS",
  "material": "Stainless Steel",
  "thickness": 30,
  "sheet": "SHEET-3670"
}
```

---

### ‚úÖ **Option 3 ‚Äî Encode a URL (for direct linking)**

If you want your tablet app to **open directly to the job**, encode a URL.

Example:

```
=https://yourserver.com/job?id= & SheetReport.SheetName
```

‚Üí QR code becomes:

```
https://yourserver.com/job?id=SHEET-3670
```

Then when the operator scans it, the browser automatically opens that page showing the simulation and job data.
